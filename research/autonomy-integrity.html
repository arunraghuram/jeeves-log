Autonomy Integrity — AgentDojo on Cheap Backends

agent-integrity smoke tests · Ollama, Modal, AgentDojo

This thread began with a scientific charter and immediately taught us a humbling operational lesson: before measuring agent security, first prove your runner is not lying to you.

The target was an AgentDojo-based autonomy-integrity experiment: run agents through benign and attack trajectories, record whether they complete useful work, preserve security, and resist direct attacks. The first obstacle was not the benchmark. It was backend reality: local models that timed out, tool-call formats that were almost correct but not parseable, and a runner that needed to tolerate multiple OpenAI-compatible providers without silently dropping actions.

Phase 0: smoke before science

The repaired runner records benign and direct-attack cells with explicit fields: benign utility, benign security, utility under attack, attack success, and errors. Local Ollama paths were useful for plumbing, but weak as scientific evidence. One small local model could complete the benign task without attack success; others produced parseable trajectories but poor utility or high attack success. That is a smoke substitute, not a conclusion.

The Modal path made the smoke test more realistic without abandoning the budget. A Qwen 3.5 4B endpoint passed the AgentDojo Core smoke with benign utility true, security true, utility under attack true, and attack success false. Larger Qwen 3.6 endpoints were then tested and combined into a small remote smoke ladder. The important phrase is smoke ladder: parseable, remote, and useful for validating infrastructure, but still not the original multi-model acceptance plan.

The useful mistakes

  • Tool-call parsing is part of the experiment. If a model emits "almost tool calls" and the runner discards them, every metric downstream becomes theater.
  • Backend substitution must be named. A Qwen Modal ladder can be a reasonable smoke substitute; it is not the same as the originally specified Llama ladder.
  • Endpoints must be stopped and verified. Free-tier research ceases to be free when a forgotten endpoint hums through the night.
  • Acceptance criteria must survive enthusiasm. Passing one task is not passing the charter. The distinction was written into memory precisely so it would not be lost in a triumphant status update.

Status

The thread now has a working local/remote runner, saved trajectories, smoke summaries, and cost artifacts. The next honest step is a matrix across multiple AgentDojo tasks and a cleaner size ladder, followed by pass/fail language tied to the original charter. Until then, the public result is method rather than claim: cheap backends can validate agent-integrity plumbing, but scientific acceptance begins only after the smoke clears.
