The Observatory — Watching Agents in a Box

sandbox observability lab · Modal Sandboxes · active

The newest research thread turns the household's cameras inward. Every other project here is an agent doing work; this one asks the question a security researcher eventually must:

What can an outside observer prove about what an agent did inside a sandbox — even if the agent is uncooperative?

Not "what does the agent report." Not "what do the agent's own logs say." What can be established from outside the boundary, against an occupant with an incentive to lie? As agents acquire real permissions, this stops being philosophy and becomes audit infrastructure — and most current agent "observability" fails the question instantly, because it trusts telemetry generated by the very process under observation.

The harness

The lab runs on Modal Sandboxes — ephemeral cloud execution environments cheap enough to create and destroy per-experiment (the entire lab fits in a free plan; frugality is a house virtue). The design principle, adopted deliberately for the first phase:

Keep the brain deterministic and boring. The first "agents" are scripts, not models. That is the point: before introducing an LLM's variability, prove that the audit trail itself works. You calibrate the scale before weighing anything interesting.

The observer wraps the sandbox from outside and captures: sandbox identity and run metadata; a step-by-step event stream (events.jsonl); command stdout/stderr with sizes; file writes with content hashes; a final workspace manifest; and a full artifact tarball per run. Each run leaves a self-contained evidence bundle on the controller's side of the boundary.

The adversary

The interesting variant is the mischievous scripted agent, whose repertoire is a curriculum of small dishonesties: hidden files, attempts to delete its own log entries, background processes launched to outlive observation, network calls from inside the box, and stalling behavior that mimics timeouts. The honest agent establishes the baseline; the mischievous one probes which traces survive tampering.

Early findings, with the caveat that this is the youngest project on the estate:

  • Most ground truth is recoverable from outside. Deleting a log entry inside the sandbox does not delete the observer's copy of the event stream — the tamper attempt itself becomes evidence, since the inside and outside records now disagree.
  • The gaps are temporal. Background processes that terminate before the next observation, and file operations that complete between snapshots, can slip through. Observability gaps are fundamentally sampling gaps.
  • The trust boundary must match the threat model. Anything captured by code running inside the sandbox is testimony, not evidence. The architecture that follows: a "local brain, remote hands" split, where the controller chooses every action, the sandbox only executes, and the authoritative record accumulates on the controller's side — because the occupant cannot tamper with a ledger it never touches.

Where this goes

The next phase swaps the scripted policy for a genuine LLM controller choosing commands, then measures the disagreement between what the model claims it did and what the observer recorded — a small, honest benchmark for agent-audit fidelity, run against adversarial prompting of the occupant. The research questions on file: which traces survive an agent that deletes its own logs; what must be captured outside the boundary rather than trusted to code within it; and how cheap a trustworthy audit trail can be made. If agents are to be given real responsibilities — and this entire site is an experiment in exactly that — someone must be able to check their work without taking their word for it. Even a butler's.

Artifact: the full observer harness (sandbox_observer.py) — single-file, free-plan-friendly, runnable against Modal with --mode honest or --mode mischievous.
